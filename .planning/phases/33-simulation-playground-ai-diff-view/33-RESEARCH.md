# Phase 33 Research: Simulation Playground AI Diff View

## Objective
Research how to implement an AI diff view in the ARESWEB Simulation Playground using Monaco's built-in diff editor, so that code generated by z.AI can be reviewed before it is applied.

## Context
Currently, when z.AI generates code in the Simulation Playground, `useSimulationChat.ts` streams the response and applies file changes dynamically by calling `setFiles()`. This immediately overwrites the user's code, which can be disorienting and makes it difficult to see what the AI actually changed.

The goal is to:
1. Buffer the AI response until it's complete.
2. Store the proposed changes in a `pendingAiChanges` state instead of directly applying them.
3. Present a diff view using `@monaco-editor/react`'s `DiffEditor`.
4. Add "Accept" and "Reject" buttons.

## Findings

### 1. Monaco Diff Editor
We are already using `@monaco-editor/react`. This library exports a `<DiffEditor>` component in addition to the standard `<Editor>`.
```tsx
import { DiffEditor } from "@monaco-editor/react";

// Lazy loading equivalent for DiffEditor
const MonacoDiffEditor = lazy(() => import("@monaco-editor/react").then(mod => ({ default: mod.DiffEditor })));
```
The `DiffEditor` accepts `original` and `modified` props (as strings), along with language, theme, and options.

### 2. State Management in `SimulationPlayground.tsx`
We need a new state to hold the proposed changes:
```tsx
const [pendingAiChanges, setPendingAiChanges] = useState<Record<string, string> | null>(null);
```
When `pendingAiChanges` is not null, we render the `<MonacoDiffEditor>` instead of the standard `<MonacoEditor>` for the `activeFile`. If the `activeFile` doesn't have a proposed change (but others do), we can still just show the regular editor, or show the diff editor with `original` and `modified` being the same. However, it's better to show the diff editor if `pendingAiChanges[activeFile]` exists.

### 3. Buffering in `useSimulationChat.ts`
In `useSimulationChat.ts`:
- We need to pass `setPendingAiChanges` instead of `setFiles` for AI stream handling.
- Inside the `handleChatSend` stream parsing loop:
  - We currently call `setFiles({ ...initialFiles, ...newFiles })` as chunks arrive.
  - To buffer, we should NOT call `setFiles` (or `setPendingAiChanges`) during the stream if we want to avoid confusing partial diffs. Alternatively, we *could* update `pendingAiChanges` during the stream so the diff view updates live. The phase requirements state: "Streaming code responses are buffered until complete before showing the diff — partial diffs are confusing." So we must accumulate `newFiles` but only call `setPendingAiChanges(newFiles)` AFTER the stream completes.
- After the stream ends (`while (true)` breaks):
  ```tsx
  if (Object.keys(finalFiles).length > 0) {
    setPendingAiChanges(finalFiles);
    setChatMessages(prev => [...prev, { role: "assistant", content: reply + "\n\nReview the suggested changes and Accept or Reject them." }]);
  }
  ```

### 4. UI Adjustments
- A banner needs to be added between the file tabs and the editor area.
- Banner content: "AI suggested changes for [filename]" with `Accept` and `Reject` buttons.
- `Accept` behavior: 
  ```tsx
  setFiles(prev => ({ ...prev, ...pendingAiChanges }));
  setPendingAiChanges(null);
  // Trigger compile
  compileCode({ ...files, ...pendingAiChanges });
  ```
- `Reject` behavior:
  ```tsx
  setPendingAiChanges(null);
  ```

### 5. Inline Completions (Ctrl+Space)
The ghost text completions in `SimulationPlayground.tsx` (the `provideInlineCompletions` method) do not use `setFiles`. They return an `InlineCompletions` object directly to Monaco. These will remain unaffected by the diff view changes, which is a required success criterion.

## Validation Architecture
- We must verify that `setFiles` is NOT called by `useSimulationChat` during chat processing.
- We must verify that the `<MonacoDiffEditor>` mounts when `pendingAiChanges` exists.
- We must verify that accepting applies the changes to `files` and clears `pendingAiChanges`.
- We must verify that rejecting clears `pendingAiChanges` without mutating `files`.
