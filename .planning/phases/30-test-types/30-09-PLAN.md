The 112 remaining violations follow these patterns:

**Pattern 1: Hono<any> (14 occurrences, 1 per file)**
```typescript
// WRONG
let testApp: Hono<any>;

// CORRECT
let testApp: Hono<TestEnv>;
```

**Pattern 2: mockExecutionContext: any (14 occurrences)**
```typescript
// WRONG
const mockExecutionContext = {
  waitUntil: vi.fn((p) => p),
} as any;

// CORRECT
import { MockExecutionContext } from "../../../src/test/types";
const mockExecutionContext: MockExecutionContext = {
  waitUntil: vi.fn((p) => p),
  passThroughOnException: vi.fn(),
};
```

**Pattern 3: res.json() as any (~40 occurrences)**
```typescript
// WRONG
const body = await res.json() as any;

// CORRECT - Import contract response type
import type { ContractResponse } from "@shared/schemas/contracts";
const body = await res.json() as ContractResponse;
// OR for simple JSON responses
const body = await res.json() as { success: boolean; [key: string]: unknown };
```

**Pattern 4: Mock function returns as any (~30 occurrences)**
```typescript
// WRONG
vi.mocked(getSessionUser).mockResolvedValueOnce({ id: "user1", role: "student" } as any);

// CORRECT - Create interface
interface MockSessionUser {
  id: string;
  role: string;
  nickname?: string;
}
vi.mocked(getSessionUser).mockResolvedValueOnce({ id: "user1", role: "student" } as MockSessionUser);
```

**Pattern 5: Mock data objects as any (~14 occurrences)**
```typescript
// WRONG
expect(mockDb.getColumn).toHaveBeenCalledWith({
  // ...
} as any);

// CORRECT - Use Partial<T> or create interface
const expectedQuery = {
  table: "tasks",
  columns: ["id", "title"],
};
expect(mockDb.getColumn).toHaveBeenCalledWith(expectedQuery);
```

</patterns>
