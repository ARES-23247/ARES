- Season selector uses a simple `<select>` with options for recent FTC seasons (2425, 2324, 2223)

Props interface:
```typescript
interface EventSelectorProps {

  selectedEventKey?: string;
}
```

<action>
Create `src/components/tools/ScoutingTool.tsx` — the main scouting tool page with 3 views:

**Layout:**
- Full-width page with ARES header pattern: title "FTC SCOUT", subtitle "AI-Powered Team Analysis & Match Predictions"
- Tab bar with 3 tabs: "Team Search", "Event Browser", "Event Analysis"

**Tab 1: Team Search**
- Number input field for FTC team number
- "Search" button → calls `scoutingApi.getTeam(number)` 
- Renders `TeamAnalysisCard` with the result
- Also fetches ranking data from current season if available

**Tab 2: Event Browser**
- `EventSelector` component at top
- When event selected: fetch and display rankings table with all teams
- Columns: Rank, Team #, Team Name, OPR, W-L-T, RP
- Clicking a team row expands to show `TeamAnalysisCard` inline

**Tab 3: Event Analysis**
- `EventSelector` at top
- "Analyze Event" button → calls `scoutingApi.analyze({ mode: "event_overview", ... })`
- Renders AI markdown response as a full event overview
- Also shows a "Predict Match" section where user picks 2 alliances (4 teams each) for match prediction

**State management:**
- `useState` for active tab, selected event, team search input, analysis results, loading states
- `useQuery` or `useEffect` for API calls with proper error handling
- Loading states show `animate-pulse` skeleton placeholders

**Error handling:**
- API errors display in `bg-ares-danger/10 border-ares-danger/30 text-ares-danger-soft` styled alerts
- Network failures show retry buttons
</action>
<acceptance_criteria>
- `src/components/tools/ScoutingTool.tsx` exists and exports default
- Component renders 3 tabs: "Team Search", "Event Browser", "Event Analysis"
- Team Search tab has number input and renders TeamAnalysisCard on search
- Event Browser tab uses EventSelector and shows rankings table
- Event Analysis tab calls AI analyze endpoint and renders markdown
- Error states handled with ARES-styled error alerts
- Loading states use skeleton/pulse animations
</acceptance_criteria>
</task>
