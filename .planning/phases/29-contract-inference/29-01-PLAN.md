## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Client → API | Untrusted HTTP requests from web clients (browser, API consumers) |
| API → Database | Trusted internal queries (Kysely parameterized queries prevent injection) |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-29-01 | Tampering | Request body validation (analytics POST endpoints) | mitigate | Zod schemas in ts-rest contracts validate request body structure and types at API boundary before data reaches handlers |
| T-29-02 | Tampering | Response data leakage | mitigate | responseValidation: true ensures handlers don't return extra fields not defined in contract, preventing accidental data exposure |
| T-29-03 | Information Disclosure | Error messages revealing internal state | mitigate | responseValidationErrorHandler returns generic error message, logs detailed error to server-side console only |

**Security-specific notes:**
- responseValidation: true ensures contract compliance even if handlers are buggy
- Zod schemas strip unknown fields by default (passthrough: false is default)
</threat_model>
