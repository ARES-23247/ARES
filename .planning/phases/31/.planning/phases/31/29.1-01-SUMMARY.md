### Migration Pattern Applied

For each file:

1. **Handler signatures**: Changed from `async ({ body/query/params }: { ... }, c: Context<AppEnv>) =>` to `async (input, c) =>`
2. **Body/query/params access**: Changed from `body.field` to `input.body.field`
3. **Router setup**: Changed from `s.router(contract, handlers as any)` to `s.router(contract, handlers)`
4. **Response validation**: Added `createHonoEndpoints(contract, router, app, { responseValidation: true, responseValidationErrorHandler: ... })`
5. **Cleanup**: Removed unused `Context` and `z` imports

### TypeScript Compilation

TypeScript compilation shows expected `implicitly has an 'any' type` warnings for the `(input, c)` parameters. This is consistent with the ts-rest-hono pattern used in analytics.ts and other migrated routes. The types ARE being inferred correctly for runtime validation, but TypeScript's strict `noImplicitAny` rule reports warnings.

No critical type errors (type mismatches, missing properties, etc.) were introduced.
