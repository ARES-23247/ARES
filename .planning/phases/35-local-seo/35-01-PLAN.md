# Plan 35-01: Local SEO & Public Discovery

## Goal
Implement structured data schema, geographical meta tags, and D1-driven dynamic sitemaps to optimize ARESWEB for local search queries like "robotics in Morgantown".

## Implementation Steps

### 1. Update `SEO.tsx` Component
- Modify `src/components/SEO.tsx` to use the `EducationalOrganization` JSON-LD schema type.
- Inject the `address` field into the JSON-LD payload to specify "Morgantown, WV".
- Ensure the `<title>` and `<meta name="description">` tags strongly reflect the team's local presence.

### 2. Update Page-Level SEO Descriptions
- In `src/pages/Home.tsx`, update the `<SEO>` description to include "Morgantown, West Virginia".
- In `src/pages/About.tsx`, similarly update the `<SEO>` description.

### 3. Expand the `sitemap.xml` Route
- Modify `functions/api/routes/sitemap.ts`.
- Add all missing static routes to the `staticRoutes` array (`/seasons`, `/outreach`, `/gallery`, `/tech-stack`, `/academy`, `/sim-runner`, `/join`, `/store`, `/leaderboard`).
- Query the `events` table (where `is_deleted = 0` and visibility allows public) and map them to `/events/:id` elements in the XML.

## Verification
- Build checking: run `npm run build` to verify no type regressions.
- Validation: Visually inspect `/sitemap.xml` output and `<head>` generation.
